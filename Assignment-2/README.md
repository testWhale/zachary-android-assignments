# Assignment 2 — 3D Fan

A 3D table fan (base, pole, hub, four spinning blades) built from a single
shared cube and rendered on Android, Windows desktop, and WebGL from one
C++ codebase. Every part is composed on the provided `Transform` push/pop
matrix stack and coloured through a `PARTCOLOR` uniform — no per-part
vertex data.

Shared source lives in `Scene/`. Your part-1-through-4 implementation is in
`Scene/Fan.h` / `Scene/Fan.cpp`. `Scene/Transform.h` / `Scene/Transform.cpp`
are provided and unmodified.

## Controls

| Gesture | Effect |
|---|---|
| Tap / click | Toggle the fan on/off (logs `Fan ON` / `Fan OFF`) |
| Drag / click-drag | Spin faster, proportional to swipe speed |
| Release | Boost decays gradually instead of snapping to base speed (fling-with-decay) |
| Keys `3` / `4` / `5` (Desktop, Web) | Change blade count at runtime |

The fan head (hub + blades) also oscillates left-right continuously; the
base and pole stay fixed.

## Prerequisites (all platforms)

- CMake 3.15+
- Git (all three build paths fetch GLM via `FetchContent` or a shallow
  clone — no manual dependency installation needed)

---

## Desktop (Windows)

**Prerequisites:** Visual Studio 2022 (Desktop C++ workload), CMake 3.15+,
internet access on first configure (downloads GLFW, GLEW, GLM).

```
script_build_and_run.bat
```

This configures `build_desktop/` with the Visual Studio 17 2022 generator,
builds the `Debug` config, and launches the exe. First run takes a few
minutes while dependencies download.

**Output:** `build_desktop\Debug\Fan3D.exe`

**Manual equivalent:**

```
cmake -S . -B build_desktop -G "Visual Studio 17 2022" -A x64
cmake --build build_desktop --config Debug
build_desktop\Debug\Fan3D.exe
```

Click to toggle the fan, click-drag to spin it up, ESC to quit.

## Android

**Prerequisites:** Android Studio (current stable), Android SDK Platform
36, NDK + CMake components (install via **SDK Manager → SDK Tools → NDK
(Side by side)** and **CMake** if Android Studio doesn't prompt for them
automatically). Min SDK 34.

1. Open the `android/` folder in Android Studio as a project.
2. Let Gradle sync. **This first sync generates `gradlew`, `gradlew.bat`,
   and `gradle/wrapper/gradle-wrapper.jar`** — commit those three once
   they appear; they're launcher scripts, not build output, and CI/other
   contributors need them.
3. Run the `app` configuration on a device or emulator with OpenGL ES 3.0
   support (**Run ▸ Run 'app'**, or the green ▶ button).

**Command line equivalent** (after the wrapper exists):

```
cd android
gradlew.bat assembleDebug
```

**Output:** `android\app\build\outputs\apk\debug\app-debug.apk`

Tap to toggle the fan, drag to spin it up. Check logs with:

```
adb logcat -s Fan3D
```

## Web (WebGL / Emscripten)

**Prerequisites:** [Emscripten SDK](https://emscripten.org/docs/getting_started/downloads.html),
activated in the current shell.

```
git clone https://github.com/emscripten-core/emsdk.git
cd emsdk
emsdk install latest
emsdk activate latest
emsdk_env.bat
```

Then, from the project root:

```
build_web.bat
```

First run clones GLM into `external/glm` (git-ignored — this is fetched,
not committed). Output is `web\index.html`.

**Serve it** (opening the file directly won't work — browsers block local
WebGL loads over `file://`):

```
cd web
python -m http.server 8080
```

Open `http://localhost:8080/` in a browser. Click to toggle, click-drag to
spin it up. Check the browser console (F12) for the `Fan ON`/`Fan OFF` and
drag-velocity log lines.

## CI/CD

`.github/workflows/build-and-deploy.yml` runs on every push/PR to `main`:

- Builds the Desktop target on `windows-latest`.
- Builds the Android debug APK on `ubuntu-latest`.
- Builds the Web target on `ubuntu-latest`.
- On push to `main` only: deploys the Web build to GitHub Pages.

The live build renders at whatever URL GitHub Pages assigns once the
workflow's `deploy-pages` job has run — see the repository's **Settings ▸
Pages** and the workflow run summary for the exact link.

## What's not in this repo

`build_desktop/`, `web/`, `external/`, and all Android/Gradle build and
IDE-cache directories are git-ignored (see `.gitignore`) — they're
generated by the steps above, not part of the submission.

## Stretch goals implemented

- **Fling with decay** — `dragBoost` isn't zeroed on release; it decays by
  2% per frame, so a hard flick keeps the fan spinning faster for a bit
  before settling back to base speed.
- **Runtime blade count** — `Fan::SetBladeCount(3|4|5)`, wired to keys
  `3`/`4`/`5` on Desktop and Web; blades re-space themselves at
  `360° / bladeCount`.
- **Oscillation** — the fan head sways ±25° around a vertical axis through
  the hub, independent of the on/off state, like a real oscillating fan.
